## Внесені зміни у вихідну логіку моделі, за варіантом:
Додано можливість вказувати кількість джерел їжі на полі. Введено показник кількості зібраної їжі у гнізді, що відображається користувачеві. Відключаємо можливість появи нових мурах за умов наявності достатньої кількості їжі у гнізді. Відображено графік зміни кількості мурах.
1. Додавання можливості задавати кількість джерел їжі
Було введено нову глобальну змінну food-sources, значення якої користувач задає через слайдер (1-3).
Процедура setup-food була змінена так, щоб створювати рівно стільки джерел їжі, скільки визначив користувач.
## Новий фрагмент коду (доданий у роботу):
;; --- Кількість джерел їжі задається користувачем ---
;; Новий слайдер: food-sources (1–3)
if (food-sources >= 1) and ((distancexy (0.6 * max-pxcor) 0) < 5)
[
  set food-source-number 1
]
if (food-sources >= 2) and ((distancexy (-0.6 * max-pxcor) (-0.6 * max-pycor)) < 5)
[
  set food-source-number 2
]

if (food-sources >= 3) and ((distancexy (-0.8 * max-pxcor) (0.8 * max-pycor)) < 5)
[
  set food-source-number 3
]
Щоб змінювати умови середовища та досліджувати вплив кількості джерел їжі на ефективність роботи колонії.

## 2. Підрахунок та відображення кількості їжі в гнізді
Було введено нову глобальну змінну nest-food, що зберігає загальний запас зібраної їжі.
У процедурі повернення мурахи до гнізда додано оновлення запасів.
## Новий фрагмент коду:
globals [
  nest-food      ;; Новий глобальний лічильник зібраної їжі
]
;; --- Додавання їжі до запасів гнізда ---
if carrying?
[
  set nest-food nest-food + 1
  set carrying? false
]
Це дозволяє контролювати ефективність колонії та визначати умови для появи нових мурах.

## 3. Можливість увімкнути або вимкнути появу нових мурах
Введено:

перемикач spawning-enabled,
слайдер food-per-new-ant, який визначає витрати їжі на одного нового мураха.
Створена нова процедура maybe-spawn-ants.
## Новий фрагмент коду (повністю доданий):
to maybe-spawn-ants
  ;; скільки нових мурах можна створити з накопиченої їжі
  let new-ants floor (nest-food / food-per-new-ant)
  if new-ants <= 0 [ stop ]

  ;; списуємо витрачену їжу
  set nest-food nest-food - new-ants * food-per-new-ant

  ;; створюємо нових мурах у зоні гнізда
  ask n-of new-ants patches with [nest?]
  [
    sprout 1
    [
      set size 2
      set color red
      set carrying? false
      rt random 360
    ]
  ]
end
А також виклик цієї процедури в go:
if spawning-enabled
[
  maybe-spawn-ants
]
Дозволяє моделювати ріст колонії залежно від її ресурсів - принцип поведінки реальних мурашників.

## 4. Додавання графіка зміни кількості мурах
Створено новий Plot, який автоматично відображає чисельність мурах.
Фрагмент коду, який оновлює графік (додається у Plot Update Commands):
plot count turtles

## Тест 1. Вплив кількості джерел їжі (food-sources)
Тестований параметр: food-sources
Мета: подивитися, як кількість джерел їжі впливає на:
швидкість накопичення nest-food,
ріст кількості мурах (якщо spawning-enabled = on).
Фіксуємо інші параметри:
population = 40
diffusion-rate = 50
evaporation-rate = 5
food-per-new-ant = 10
spawning-enabled = on (вмикаємо)
500 тіків
Робимо 3 запускі моделі:
food-sources = 1/2/3

1. Порівняння результатів (перші 500 тiків)
## Випадок 1: food-sources = 1
Початок: 40 мурах.
Приблизно до ~130-го тіку чисельність майже не змінюється (все ще 40).
Потім починається повільне зростання.
На 500-му тіку: ≈ 50 мурах.
Популяція зросла на ~10 мурах за 500 тактів.

## Випадок 2: food-sources = 2
Початок: 40 мурах.
Зростання починається раніше, десь із ~97-го тіку.
Популяція зростає більш стабільно й довше.
На 500-му тіку: ≈ 59 мурах.
Приріст приблизно +19 мурах - найвище значення з трьох експериментів.

## Випадок 3: food-sources = 3
Початок: 40 мурах.
Зростання стартує ще раніше (приблизно з ~74-го тіку).
Популяція піднімається до ≈ 52 мурах і потім тримається біля цього рівня.
На 500-му тіку: ≈ 52 мурах.
Приріст приблизно +12 мурах.

За результатами моделювання протягом перших 500 тактів для різної кількості джерел їжі було отримано такі значення чисельності популяції мурах:
при 1 джерелі їжі чисельність зросла з 40 до приблизно 50 особин;
при 2 джерелах їжі чисельність зросла з 40 до приблизно 59 особин;
при 3 джерелах їжі чисельність зросла з 40 до приблизно 52 особин.

Це означає, що збільшення кількості джерел їжі загалом сприяє зростанню популяції, однак максимальний приріст у даних умовах спостерігається не при трьох, а при двох джерелах.

Можливе пояснення:
при 1 джерелі ресурсу недостатньо, мурахи довго шукають їжу, тому темп розмноження низький;
при 3 джерелах їжа більш розсіяна по полю, мурахи та феромонні доріжки “розмазуються” між кількома напрямками, і ефективність використання кожного джерела знижується;
при 2 джерелах досягається компроміс між доступністю ресурсу та концентрацією мурах на шляхах, тому саме в цьому випадку популяція зростає найактивніше.
